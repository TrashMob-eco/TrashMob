## ?? For Command File Authors

### Creating a New Command File

1. **Name it:** `.claude_[purpose]_command.md`
2. **Start with purpose statement**
3. **Use numbered steps**
4. **Include code/markdown examples**
5. **Add quality checks**
6. **Provide user summary template**
7. **Update this index file**

### Command File Best Practices

? **Do:**
- Use clear, imperative language ("Create file", "Update section")
- Provide complete file paths
- Include code snippets
- Show before/after examples
- Add quality verification steps
- Reference related commands

? **Don't:**
- Assume context (be explicit)
- Skip error handling
- Forget cross-references
- Use vague language
- Omit examples

## ?? Troubleshooting

### AI Assistant Can't Find Command
**Problem:** AI references wrong file or section  
**Solution:** Update this index with correct links; use consistent naming

### Command Produces Inconsistent Results
**Problem:** Different AIs interpret steps differently  
**Solution:** Add more explicit instructions; include examples; add verification steps

### Command Missing Information
**Problem:** User needs something not covered  
**Solution:** Create new command or add section to existing file; update this index

### Cross-References Break
**Problem:** Links to other files don't work  
**Solution:** Use relative paths; verify links after creation; add link validation to quality checks
